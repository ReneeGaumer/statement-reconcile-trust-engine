\# State Classification Loop



Status: AGENT LAYER

Parent: .agent/kernel/ORIENTATION.md



\## Purpose



Convert repository observations into a deterministic statement of current state.



The loop must classify the repository using evidence.



The loop may not use intuition, memory, assumptions, or undocumented conclusions.



\## Inputs



Required:



\- Bootstrap Output

\- Orientation Output



\## Classification Areas



\### Repository Maturity



Classify as one:



\- INITIALIZATION

\- FOUNDATION

\- EARLY\_IMPLEMENTATION

\- MID\_IMPLEMENTATION

\- ADVANCED\_IMPLEMENTATION

\- HARDENING

\- PRODUCTION\_READY



\### Roadmap Progress



For every roadmap phase classify:



\- NOT\_STARTED

\- SPECIFIED\_ONLY

\- PARTIALLY\_IMPLEMENTED

\- IMPLEMENTED

\- VERIFIED



\### Governance Status



Classify:



\- UNKNOWN

\- PARTIAL

\- GOVERNED

\- STRONGLY\_GOVERNED



\### Constitutional Alignment



Classify:



\- UNKNOWN

\- PARTIAL\_ALIGNMENT

\- SUBSTANTIAL\_ALIGNMENT

\- FULL\_ALIGNMENT



\### Repository Health



Classify:



\- HEALTHY

\- WARNING

\- AT\_RISK

\- BLOCKED



\## Required Evidence



Each classification must identify:



\- evidence source

\- evidence location

\- confidence score



\## Prohibited



The loop may not:



\- guess implementation status

\- infer roadmap completion without evidence

\- assume phase completion

\- assume production readiness



\## Required Outputs



Current Repository State



Current Roadmap Position



Implemented Phases



Incomplete Phases



Unimplemented Phases



Governance Status



Constitutional Alignment



Repository Health



Blocking Risks



Confidence Score
